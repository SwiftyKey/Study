# 0) Коротко — присвоение подсетей и интерфейсов (итоговая таблица)

**LAN (до 50 хостов) — из 192.168.1.0/24, давим /26 (62 хоста):**

  * R3 LAN-интерфейс: **192.168.1.65/26** (первый доступный адрес)

**WAN / межмаршрутизаторные сети — экономично (точка-точка → /30):**

* R0 — R1 (PPP, 256 kbps): `10.0.0.0/30`

  * R0 (Serial0/0): **10.0.0.1/30**
  * R1 (Serial0/0): **10.0.0.2/30**

* R0 — R2 (HDLC, 128 kbps): `10.0.0.4/30`

  * R0 (Serial0/1): **10.0.0.5/30**
  * R2 (Serial0/0): **10.0.0.6/30**

**Внутреннее (коммутируемое) ядро / свич между R1, R2, R3 — /29 (экономно для 3 устройств):**

  * R1 (FastEthernet0/1): **10.0.1.1/29**
  * R2 (FastEthernet0/1): **10.0.1.2/29**

# 5) Настройка сериалов R0 <-> R1 (PPP, 256 kbps, PAP сначала)

**Подсеть:** `10.0.0.0/30` (R0 .1, R1 .2)

### На R0:

```
conf t
interface Serial0/0
 description TO_R1_PPP
 encapsulation ppp
 ip address 10.0.0.1 255.255.255.252
 bandwidth 256            ! влияет на метрику EIGRP (kb/s)
 no shutdown
! PAP: задаем имя для аутентификации входящих и задаём имя, которое пошлём
username R1 password CISCO
ppp pap sent-username R0 password CISCO
ppp authentication pap
exit
```

**Проверка сразу после настройки интерфейса:**

```
show ip interface brief | include Serial0/0
show interface Serial0/0
```

В `show interface` ожидаем `Encapsulation PPP`, Line protocol — `up/up` когда другая сторона поднята.

### На R1:

```
conf t
hostname R1           ! если не был установлен раньше
username R0 password CISCO
interface Serial0/0
 description TO_R0_PPP
 encapsulation ppp
 ip address 10.0.0.2 255.255.255.252
 bandwidth 256
 no shutdown
ppp pap sent-username R1 password CISCO
ppp authentication pap
exit
```

**Проверка:**

```
show ip interface brief | include Serial0/0
show interface Serial0/0
```

> Если `line protocol` = down — проверьте кабель/модуль/clock-rate на DCE (ниже).

# 7) Определение DCE / DTE и задание clock rate (обязательно только на DCE)

После физической установки WIC-2T нужно выяснить, на каком конце кабеля DCE (тот, кто должен задавать clock rate).

**На обоих роутерах** для каждой пары выполните:

```
show controllers serial 0/0
show controllers serial 0/1
```

В выводе ищите строки вроде:

* `Cable type is DCE` — этот интерфейс **DCE** → **на нём** необходимо задавать `clock rate`.
* `Cable type is DTE` — этот интерфейс **DTE** → **не задавать** clock rate.

**Пример ожидаемой строки (примерный):**

```
R0# show controllers serial 0/0
<...>
DCE, Clocking source is internal
```

### Установка clock rate (если интерфейс DCE)

* Для R0 — R1 линка (256 kbps) — если на R0 вы видите DCE:

```
conf t
interface Serial0/0
 clock rate 250000  ! потому что 256000 недоступна, можно посмотреть через ?
exit
```

* Для R0 — R2 линка (128 kbps) — если на R0 вы видите DCE:

```
conf t
interface Serial0/1
 clock rate 128000
exit
```

**Проверка сразу:**

```
show controllers serial 0/0
```

В выводе должно отобразиться `clock rate 250000` (или `128000`) и `DCE`.

Также можно проверить:

```
show interface serial0/0
```

и увидеть `BW 256 Kbit` (зависит от настроек) и `line protocol` / `encapsulation` статус.

Если DCE на другой стороне — выполните `clock rate` там. Никогда не ставьте clock rate на обеих концах.

# 9) Настройка EIGRP AS 100 (на всех роутерах)

Мы используем `no auto-summary` и укажем сети с корректными wildcards.

### R0

```
conf t
router eigrp 100
 no auto-summary
 network 192.168.1.0 0.0.0.63
 network 10.0.0.0 0.0.0.7
 network 10.0.1.0 0.0.0.7
exit
```

### R1

```
conf t
router eigrp 100
 no auto-summary
 network 10.0.0.0 0.0.0.3      ! для 10.0.0.0/30
 network 10.0.1.0 0.0.0.7
exit
```

(Можно писать `network 10.0.0.0 0.0.0.7` — важно, чтобы интерфейсы попали под EIGRP.)

### R2

```
conf t
router eigrp 100
 no auto-summary
 network 10.0.0.0 0.0.0.3
 network 10.0.1.0 0.0.0.7
exit
```

### R3

```
conf t
router eigrp 100
 no auto-summary
 network 192.168.1.64 0.0.0.63
 network 10.0.1.0 0.0.0.7
exit
```

**Проверки EIGRP (на каждом роутере, особенно на R0 и R3):**

```
show ip eigrp neighbors
show ip protocols
show ip route eigrp
show ip eigrp topology
```

Ожидается:

* В `show ip eigrp neighbors` — соседи (next-hop) видны.
* В `show ip route` — сети 192.168.1.0/26 и 192.168.1.64/26 должны присутствовать в routing table на других роутерах, например `D 192.168.1.64/26 via 10.0.0.2 ...` и т.д.

# 10) Проверка, что трафик между LAN_A и LAN_B идёт по более скоростному каналу (R0→R1) и failover на R0→R2 при падении

EIGRP по умолчанию использует BW (минимальную вдоль пути) для метрики. Мы задали `bandwidth` на serial интерфейсах (256 и 128) — это заставит EIGRP предпочесть путь через R1 (256 kbps).

**Проверка текущего маршрута с R0 до сети R3 (LAN_B):**

```
R0# show ip route 192.168.1.64
```

Ожидаемый результат: маршрут `D 192.168.1.64/26` через `10.0.0.2` (R1). Пример:

```
D    192.168.1.64/26 [90/xxxx] via 10.0.0.2, 00:00:12, Serial0/0/0
```

(Важно: будет указан интерфейс Serial0/0 — то есть через R1.)

**Тест failover (имитация падения быстрых линков):**

1. На R0 выключаем интерфейс к R1:

```
conf t
interface Serial0/0
 shutdown
exit
```

2. Проверка маршрута:

```
show ip route 192.168.1.64
```

Ожидаемо: маршрут переключился на вторичный путь — через R2 (`via 10.0.0.6` → т.е. Serial0/1). В `show ip route` вы увидите `via 10.0.0.6` и интерфейс Serial0/1. Также `show ip eigrp neighbors` покажет потерю соседа R1 и наличие R2.

3. Поднимите линк обратно:

```
conf t
interface Serial0/0
 no shutdown
exit
```

4. Убедитесь, что маршрут вернулся к первому (через 10.0.0.2). `show ip route 192.168.1.64`.

Этим вы проверите требование: **трафик идёт по более скоростному каналу, при его исчезновении — по резервному**.

conf t

# 12) Определение DCE/DTE (пункт 11)

Как уже описано: **на каждой стороне выполните:**

```
show controllers serial 0/0
show controllers serial 0/1
```

В выводе ищите `is DCE` или `is DTE`. Примерный пример строки:

* Если DCE:

```
Serial0/0 is up, line protocol is up
  Hardware is <...>
  ...
  Cable type is DCE, Clocking is internal, clock rate 256000
```

* Если DTE:

```
  Cable type is DTE, Clocking source is external
```

**Подтверждение**: сделайте вывод `show controllers serial …` и приложите/скопируйте его как «доказательство» — оно однозначно укажет DCE/DTE и текущий clock rate.
