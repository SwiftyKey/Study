### 📌 **Подключение устройств (кабели и порты)**

#### 1. **Frame Relay Cloud → Router_A и Router_C**
- **Frame Relay Cloud** — это специальное устройство в PT.
- Подключите его к **Router_A** и **Router_C** через **Serial** порты.
- Используйте **Serial DCE** кабель (в PT: `Serial DCE` или `Serial` — зависит от версии).
- На Frame Relay Cloud:
  - Нажмите на него → Config → Interface → выберите `Serial0/0` для Router_A, `Serial0/1` для Router_C.
  - Укажите **DLCI 100** для обоих соединений (как в схеме).

> ✅ **Как подключить Frame Relay в PT**:
> - Выберите устройство "Cloud" → вкладка "Config" → Interface → добавьте Serial интерфейсы.
> - Для каждого интерфейса укажите DLCI = 100.
> - Подключите к роутерам через Serial порты (например, `Serial0/0` на Router_A и Router_C).

#### 2. **Router_C → Router_B (PPP)**
- Подключите **Router_C** и **Router_B** через **Serial** порты.
- Используйте **Serial DCE** кабель (на одном из роутеров должен быть DCE — обычно на Router_C).
- На Router_C: `Serial0/1` → Router_B: `Serial0/0`

#### 3. **Router_A → Switch1**
- Подключите **Router_A** к **Switch1** через **FastEthernet** порт (например, `Fa0/0` на роутере → `Fa0/1` на свиче).

#### 4. **Router_B → Switch2**
- Подключите **Router_B** к **Switch2** через **FastEthernet** порт (например, `Fa0/0` → `Fa0/1`).

#### 5. **Switch1 → Switch2**
- Подключите через **FastEthernet** (например, `Fa0/2` на SW1 → `Fa0/2` на SW2).

#### 6. **Switch1 → Switch3**
- Подключите через **FastEthernet** (например, `Fa0/3` на SW1 → `Fa0/1` на SW3).

#### 7. **Switch2 → Switch3**
- Подключите через **FastEthernet** (например, `Fa0/3` на SW2 → `Fa0/2` на SW3).

#### 8. **PC_A → Switch3**
- Подключите через прямой кабель.
- PC_A → `FastEthernet0` → Switch3 → `Fa0/3`

#### 9. **PC_B → Switch3**
- Подключите через прямой.
- PC_B → `FastEthernet0` → Switch3 → `Fa0/4`

#### 10. **Server_C → Router_C**
- Подключите через кроссоверный.
- Server_C → `FastEthernet0` → Router_C → `Fa0/1`

## 🎯 **ШАГ 2: СОЗДАНИЕ VLAN A и B**

### Цель:
- VLAN A: PC_A, Router_A
- VLAN B: PC_B, Router_B

> ⚠️ Внимание: Роутеры подключены к свичам через транковые порты (для VLAN). Мы будем использовать subinterfaces.

### 🔧 Команды:

```bash
enable

vlan 20
 name VLAN_B

```

#### **На Switch2 (аналогично):**
```bash
enable
configure terminal
vlan 10
 name VLAN_A
vlan 20
 name VLAN_B
exit
```

#### **На Switch3 (не нужно создавать — получит от сервера):**
```bash
show vlan brief
```
Убедитесь, что VLAN 10 и 20 есть.

---

### 🔄 Настройка портов на свичах:

#### **Switch1:**
- Порт к Router_A → транк (Fa0/0)
- Порт к Switch2 → транк (Fa0/2)
- Порт к Switch3 → транк (Fa0/3)

```bash
interface FastEthernet0/0
 switchport mode trunk
 switchport trunk allowed vlan 10,20
 no shutdown

interface FastEthernet0/2
 switchport mode trunk
 switchport trunk allowed vlan 10,20
 no shutdown

interface FastEthernet0/3
 switchport mode trunk
 switchport trunk allowed vlan 10,20
 no shutdown
```

#### **Switch2:**
- Порт к Router_B → транк (Fa0/0)
- Порт к Switch1 → транк (Fa0/2)
- Порт к Switch3 → транк (Fa0/3)

```bash
interface FastEthernet0/0
 switchport mode trunk
 switchport trunk allowed vlan 10,20
 no shutdown

interface FastEthernet0/2
 switchport mode trunk
 switchport trunk allowed vlan 10,20
 no shutdown

interface FastEthernet0/3
 switchport mode trunk
 switchport trunk allowed vlan 10,20
 no shutdown
```

#### **Switch3:**
- Порт к PC_A → access VLAN 10 (Fa0/3)
- Порт к PC_B → access VLAN 20 (Fa0/4)
- Порты к SW1 и SW2 → транк

```bash
interface FastEthernet0/3
 switchport mode access
 switchport access vlan 10
 no shutdown

interface FastEthernet0/4
 switchport mode access
 switchport access vlan 20
 no shutdown

interface FastEthernet0/1
 switchport mode trunk
 switchport trunk allowed vlan 10,20
 no shutdown

interface FastEthernet0/2
 switchport mode trunk
 switchport trunk allowed vlan 10,20
 no shutdown
```

---

### ✅ ПРОВЕРКА VLAN:

На любом свиче:
```bash

show interfaces trunk
```
Убедитесь, что:
- VLAN 10 и 20 существуют.
- Порты правильно назначены (access/trunk).
- Транки передают VLAN 10 и 20.

## 🎯 **ШАГ 8: НАСТРОЙКА EIGRP AS 10 С УЧЕТОМ ПРОПУСКНОЙ СПОСОБНОСТИ**

### Цель:
- Все роутеры объявляют свои сети через EIGRP AS 10.
- Метрика должна учитывать реальную пропускную способность (bandwidth).

> ⚠️ По умолчанию EIGRP использует bandwidth и delay. Мы установим bandwidth вручную, чтобы метрика была корректной.

---

### 🔧 Команды:

#### **На Router_A:**
```bash
router eigrp 10
 network 192.168.1.64 0.0.0.31
 network 192.168.1.112 0.0.0.3
 no auto-summary
 exit
```

#### **На Router_B:**
```bash
router eigrp 10
 network 192.168.1.96 0.0.0.15
 network 192.168.1.116 0.0.0.3
 no auto-summary
 exit
```

#### **На Router_C:**
```bash
router eigrp 10
 network 192.168.1.0 0.0.0.63
 network 192.168.1.112 0.0.0.3
 network 192.168.1.116 0.0.0.3
 no auto-summary
 exit
```

> ⚠️ Убедитесь, что на всех роутерах `bandwidth` установлено правильно (мы уже сделали это при настройке интерфейсов).

---

### ✅ ПРОВЕРКА EIGRP:

На любом роутере:
```bash
show ip eigrp neighbors
show ip route eigrp
show ip protocols
```

→ Должны быть соседи (2 соседа на Router_C, по одному на Router_A и Router_B).

→ В таблице маршрутизации должны быть все сети:
- 192.168.1.0/26 (C)
- 192.168.1.64/27 (A)
- 192.168.1.96/28 (B)
- 192.168.1.112/30 (WAN A-C)
- 192.168.1.116/30 (WAN B-C)
