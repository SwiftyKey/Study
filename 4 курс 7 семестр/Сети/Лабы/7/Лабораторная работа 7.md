# 1. Сборка в Packet Tracer (физически)

Соединяем кабелями:

* SW1 Fa0/1 <-> SW2 Fa0/1
* SW1 Fa0/2 <-> SW2 Fa0/2
* SW1 Fa0/3 <-> SW3 Fa0/3
* SW2 Fa0/3 <-> SW4 Fa0/3
* SW3 Fa0/2 <-> SW4 Fa0/2
* PC1 <-> SW3 Fa0/1
* PC2 <-> SW4 Fa0/1

### Проверка после шага:

В Packet Tracer должен загореться **зелёный (или оранжевый-ожидание)** индикатор линка. Если линк красный → значит кабель/порт неправильный.

# 3. Базовая настройка коммутаторов

Пример для SW1 (на всех аналогично):

```text
enable
configure terminal
hostname SW1
no ip domain-lookup
spanning-tree mode pvst
end
write memory
```

(На SW2 → hostname SW2, на SW3 → SW3, на SW4 → SW4.)

### Проверка после шага:

На любом свитче:

```text
show spanning-tree
```

Должен отобразиться VLAN 1, даже если root пока выбран случайным образом.

# 4. EtherChannel и Root Bridge

## SW1:

```text
conf t
port-channel load-balance dst-ip
interface range Fa0/1 - 2
 switchport trunk encapsulation dot1q
 switchport mode trunk
 channel-group 1 mode active
exit
interface Port-channel1
 switchport trunk encapsulation dot1q
 switchport mode trunk
exit
spanning-tree vlan 1 root primary
end
write memory
```

## SW2:

```text
conf t
port-channel load-balance dst-ip
interface range Fa0/1 - 2
 switchport trunk encapsulation dot1q
 switchport mode trunk
 channel-group 1 mode active
exit
interface Port-channel1
 switchport trunk encapsulation dot1q
 switchport mode trunk
exit
spanning-tree vlan 1 root secondary
end
write memory
```

### Проверка после шага:

На SW1 или SW2:

```text
show etherchannel summary
```

Ожидаем — `Po1` в состоянии `(P)` (participating).

```text
show spanning-tree vlan 1
```

На SW1 будет `This bridge is the root`.

# 5. Настройка SW3 и SW4 (транки + PortFast)

### SW3:

```text
conf t
hostname SW3
no ip domain-lookup
spanning-tree mode pvst

interface Fa0/2
 switchport mode trunk
exit

interface Fa0/3
 switchport mode trunk
exit

interface Fa0/1
 switchport mode access
 switchport access vlan 1
 spanning-tree portfast
exit
end
write memory
```

### SW4:

```text
conf t
hostname SW4
no ip domain-lookup
spanning-tree mode pvst

interface Fa0/2
 switchport mode trunk
exit

interface Fa0/3
 switchport mode trunk
exit

interface Fa0/1
 switchport mode access
 switchport access vlan 1
 spanning-tree portfast
exit
end
write memory
```

### Проверка после шага:

На SW3 и SW4:

```text
show spanning-tree
```

Должно быть: Root Port (к SW1 или SW2), Designated Port, и один порт в Blocking (между SW3 и SW4).

# 6. Проверка BPDU

В Packet Tracer включить **Simulation Mode**, фильтр STP. Запустить трафик.
Ожидаем кадры к адресу `01-00-0C-CC-CC-CD` (PVST).

У меня в половине случаев встречался вот такой адрес `0180.C200.0000`, почему так, я не разобрался, есть только вот такое объяснение

```text
DEST ADDR: 0100.0CCC.CCCD — это реальный мультикаст PVST+ (Per-VLAN Spanning Tree) для VLAN 1.

Именно этот адрес используется в железе Cisco для STP кадра.

В Packet Tracer он может показываться только на Root Bridge, когда BPDU реально уходит в сеть.

Почему раньше был 0180.C200.0000:

Packet Tracer в Simulation Mode иногда отображает упрощённый «виртуальный» мультикаст для BPDU, чтобы было наглядно.

Но если поймать кадр с Root Bridge, PT показывает правильный адрес PVST+ 0100.0CCC.CCCD.
```

# Вопрос 1. Роли и статусы портов в STP
## Роли портов
1. Root Port (RP)

Порт на некорневом коммутаторе, обеспечивающий наилучший (самый дешёвый) путь к корневому мосту. У каждого некорневого коммутатора —
только один Root Port.

2. Designated Port (DP)

Порт, который отправляет трафик в сегмент сети (например, в сегмент между двумя коммутаторами). На каждом сегменте должен быть ровно один Designated Port. Корневой мост имеет DP на всех своих активных портах.

3. Alternate Port (AP) или Blocked Port (BP)

Резервный порт, ведущий к корневому мосту через другой путь. Блокируется, чтобы избежать петли. Используется как резервный путь, если основной (Root Port) выйдет из строя.

4. Backup Port

Резервный порт в сегменте, где уже есть Designated Port (редко встречается — например, при подключении хаба или петле на одном коммутаторе). Также заблокирован.

## Статусы портов

1. Disabled: Порт выключен (shutdown)
2. Blocking: Порт не участвует в пересылке, но слушает BPDU
3. Listening: Переходное состояние: строит активную топологию
4. Learning: Изучает MAC-адреса, но
ещё не передаёт данные
5. Forwarding: пересылает данные и BPDU
