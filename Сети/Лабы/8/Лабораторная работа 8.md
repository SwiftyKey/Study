# Топология и обозначение портов

* SW1 Fa0/1 <-> SW2 Fa0/1  (транк)
* SW1 Fa0/2 <-> SW3 Fa0/1  (транк)
* SW2 Fa0/2 <-> SW3 Fa0/2  (транк)
* PC1 <-> SW1 Fa0/3  (access → VLAN 2)
* PC3 <-> SW2 Fa0/3  (access → VLAN 3)
* PC2 <-> SW3 Fa0/3  (access → VLAN 3)
* PC4 <-> SW3 Fa0/4  (access → VLAN 2)


# План адресации

VLAN 2 (ACCOUNTERS) — сеть `192.168.1.0/25` (маска `255.255.255.128`)

* PC1: `192.168.1.10/25`, GW (опционально): `192.168.1.1`
* PC4: `192.168.1.20/25`, GW (опционально): `192.168.1.1`

VLAN 3 (PROGRAMMERS) — сеть `192.168.1.128/25` (маска `255.255.255.128`)

* PC2: `192.168.1.130/25`, GW (опционально): `192.168.1.129`
* PC3: `192.168.1.140/25`, GW (опционально): `192.168.1.129`


# Шаг 1. Конфиг на SW1

Заходим в CLI SW1:

```
enable
configure terminal
hostname SW1

! VTP
vtp domain POVT
vtp mode server
vtp password POVTpass

! создаём VLANы
vlan 2
 name ACCOUNTERS
exit
vlan 3
 name PROGRAMMERS
exit

! Настроим интерфейсы транков к другим свичам
interface FastEthernet0/1
 description TO_SW2
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 2,3
 no shutdown
exit

interface FastEthernet1/1
 switchport trunk encapsulation dot1q
 switchport mode trunk
 no shutdown
exit

! Порт к PC1 — access VLAN 2 + PortFast
interface FastEthernet0/3
 description TO_PC1
 switchport mode access
 switchport access vlan 2
 spanning-tree portfast
 switchport nonegotiate
 no shutdown
exit

! Сделаем SW1 корневым для VLAN 2
spanning-tree vlan 2 root primary

! (опционально) уменьшить priority вручную:
! spanning-tree vlan 2 priority 4096

! Сохраним конфиг
end
write memory
```

### Проверки на SW1

```
show vtp status
 show output должен: Domain Name = POVT, Mode = Server, Configuration revision > 0

show vlan brief
 show VLAN 2 и 3 должны присутствовать и Fa0/3 быть в VLAN 2

show interfaces trunk
 должен показать Fa0/1 и Fa0/2 как trunk, Allowed VLANs = 2,3

show interfaces FastEthernet0/3 switchport
 покажет, что интерфейс в mode: access, Access VLAN:2, Portfast: enabled

show spanning-tree vlan 2
 покажет что Root for VLAN 2 — SW1 (Bridge ID = его MAC, Root ID совпадает)
```


# Шаг 2. Конфиг на SW2

```
enable
configure terminal
hostname SW2
vtp domain POVT
vtp mode transparent
vtp password POVTpass
interface FastEthernet0/1
 description TO_SW1
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 2,3
 no shutdown
exit
interface FastEthernet0/2
 description TO_SW3
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 2,3
 no shutdown
exit
interface FastEthernet1/1
 switchport mode access
 spanning-tree portfast
 switchport nonegotiate
 no shutdown
exit
spanning-tree vlan 3 root primary
end
write memory
```

### Проверки на SW2

```
show vtp status
  Domain Name = POVT, Mode = Transparent (transparent не принимает/распространяет DB, но должен показывать домен и пароль)

show interfaces trunk
  Fa0/1 и Fa0/2 — trunk, Allowed VLANs = 2,3

show vlan brief
  Fa0/3 — в VLAN 3

show spanning-tree vlan 3
  Root for VLAN 3 — SW2 (Bridge ID = MAC SW2)
```

```
show interfaces FastEthernet0/4 switchport
  Access VLAN: 2, Portfast: enabled
```

# Шаг 4. Какой адрес использует VTP для рассылки?

Ответ: VTP  рассылки отправляются по каналу уровня 2 через trunk и используют **Ethernet multicast MAC-адрес**:

**`01-00-0C-CC-CC-CC`**

Также VTP-адр. рассылки передаются в нативной VLAN (обычно VLAN 1) и в 802.1Q или ISL кадре (в зависимости от encapsulation).

Проверить, можно ли увидеть VTP пакеты: на реальном оборудовании / в эмуляторе — через сниффер на trunk-интерфейсе (WireShark) увидеть кадры с dst MAC `01:00:0C:CC:CC:CC`. (В Packet Tracer прямой сниффер нет, но `show vtp status` + `show interfaces trunk` и правильные домен/пароль подтвердят работу.)

# Шаг 6. Проверки связи (ping)

1. Проверим связь внутри VLAN ACCOUNTERS (VLAN 2):

```
PC1> ping 192.168.1.20   ! PC1 -> PC4
```

Ожидаемый результат: **успех (TTL, время)** — т.к. оба в VLAN 2.

2. Проверим связь внутри VLAN PROGRAMMERS (VLAN 3):

```
PC2> ping 192.168.1.140  ! PC2 -> PC3
```

Ожидаемый результат: **успех**.

3. Проверим связь между VLAN-ами (PC1 -> PC3):

```
PC1> ping 192.168.1.140  ! PC1 (VLAN2) -> PC3 (VLAN3)
```

Ожидаемый результат: **неудача (Destination host unreachable / Request timed out)** — пока не настроен маршрутизатор/интер-VLAN routing.
